# 커밋 컨벤션

## 형식

```
tag: 한글 설명

(선택) 왜 이렇게 했는지에 대한 설명
```

## 태그 목록
- `feat` — 새로운 기능 추가
- `fix` — 버그 수정
- `refactor` — 동작 변경 없는 코드 구조 개선
- `docs` — 문서 추가/수정
- `test` — 테스트 추가/수정
- `chore` — 빌드, 설정 등 기타 변경

## 규칙
- 태그는 영어 소문자, 설명은 한글로 작성한다
- 제목은 간결하게, 무엇을 했는지 명확히 드러낸다
- 바디는 필요할 때만 작성하며, **왜** 그렇게 했는지를 설명한다
- 하나의 커밋은 하나의 작업 단위를 담는다
